# Task 4: Google Play Store Data Analysis

Level: 2

## 📌 Project Overview
The objective of this project was to clean, analyze, and visualize Google Play Store data to extract insights into the Android app market. This involved handling a highly messy dataset and performing Natural Language Processing (NLP) sentiment analysis on user reviews.

## 🛠️ Tech Stack
* **Python:** Scripting and logic.
* **Pandas & NumPy:** Data cleaning (Regex/string manipulation) and merging datasets.
* **Matplotlib & Seaborn:** Static data visualization.
* **Plotly:** Advanced interactive data visualization.

## 🧠 Key Insights & Process
1. **Data Cleaning:** Cleaned the raw data by removing string characters (`$`, `+`, `,`, `M`, `k`) from the Price, Installs, and Size columns to convert them into usable numerical formats.
2. **Category Exploration:** Identified that Family, Games, and Tools are the most saturated app categories.
3. **Metrics Analysis:** Discovered that **Paid apps** have a slightly higher average rating (4.26) than **Free apps** (4.17).
4. **Sentiment Analysis:** Merged app data with user reviews to visualize emotional trends, revealing that 61% of user reviews left on the Play Store are strictly Positive.
5. **Interactive Visualization:** Built a dynamic, interactive bubble chart using Plotly to explore the top-performing apps by rating, reviews, and installs simultaneously.
